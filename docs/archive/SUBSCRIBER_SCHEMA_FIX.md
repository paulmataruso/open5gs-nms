# Subscriber Schema Updates - March 6, 2026

## Summary

Updated the subscriber implementation to match the exact Open5GS MongoDB schema based on analysis of a real MongoDB backup file.

---

## Key Schema Corrections

### 1. **UE Address Field Names**
**Problem:** Used incorrect field names for UE addresses  
**Before:** `ue.addr` and `ue.addr6`  
**After:** `ue.ipv4` and `ue.ipv6`

**Impact:** This was a critical fix - the old field names would not have worked with actual Open5GS deployments.

### 2. **Security Fields**
**Added:**
- `security.op` - Separate OP field (can be null, different from OPc)
- `security.sqn` - Changed from `string` to `number` type

**From MongoDB:**
```json
"security": {
  "k": "58687C3B3F77695E201DC5078A899C77",
  "amf": "8000",
  "op": null,
  "opc": "1F93B56309D32808455C1EDDEA6E9CAE",
  "sqn": 152998
}
```

### 3. **MME Integration Fields**
**Added fields for 4G/LTE MME integration:**
- `mme_host` (string | string[]) - MME hostname
- `mme_realm` (string | string[]) - MME realm for diameter
- `purge_flag` (boolean | boolean[]) - UE detach purge flag
- `mme_timestamp` (number) - Last MME update timestamp

**Example from real data:**
```json
"mme_host": "mme.localdomain",
"mme_realm": "localdomain",
"purge_flag": false,
"mme_timestamp": 1771716117683499
```

### 4. **Access Restriction Data**
**Correction:** Updated default value  
**Default value:** `32` (not `0`)

This is the standard Open5GS default for access restriction.

### 5. **IMEISV Field Type**
**Changed:** From `string` to `string | string[]`  
**Reason:** Open5GS supports both single IMEISV and arrays

### 6. **MongoDB _id Fields**
**Added `_id` fields to:**
- `SubscriberSlice._id`
- `SubscriberSession._id`

These are auto-generated by MongoDB and should be preserved.

## Complete Subscriber Schema (From Real Data)

```typescript
interface Subscriber {
  _id: ObjectId;
  imsi: string;
  msisdn: string[];
  imeisv: string | string[];
  
  // MME Integration (4G/LTE)
  mme_host?: string | string[];
  mme_realm?: string | string[];
  purge_flag?: boolean | boolean[];
  mme_timestamp?: number;
  
  // Security
  security: {
    k: string;           // 128-bit key (32 hex chars)
    op?: string | null;  // Operator variant key
    opc: string;         // Operator variant key (derived)
    amf: string;         // Authentication management field (4 hex chars)
    sqn?: number;        // Sequence number (auto-incremented)
  };
  
  // Subscriber Status (TS 29.272)
  subscriber_status: number;           // 0=SERVICE_GRANTED, 1=OPERATOR_DETERMINED_BARRING
  operator_determined_barring: number; // Bitmask
  network_access_mode: number;         // 0=PACKET_AND_CIRCUIT, 2=ONLY_PACKET
  access_restriction_data: number;     // Default: 32
  subscribed_rau_tau_timer: number;    // Minutes
  
  // AMBR
  ambr: {
    uplink: { value: number; unit: number };
    downlink: { value: number; unit: number };
  };
  
  // Slices
  slice: Array<{
    _id?: string;
    sst: number;
    sd?: string;
    default_indicator?: boolean;
    session: Array<{
      _id?: string;
      name: string;      // DNN/APN
      type: number;      // 1=IPv4, 2=IPv6, 3=IPv4v6
      
      // QoS
      qos: {
        index: number;   // 5QI/QCI
        arp: {
          priority_level: number;           // 1-15
          pre_emption_capability: number;   // 1=enabled, 2=disabled
          pre_emption_vulnerability: number; // 1=enabled, 2=disabled
        };
      };
      
      // Session AMBR
      ambr: {
        uplink: { value: number; unit: number };
        downlink: { value: number; unit: number };
      };
      
      // UE Addresses (CRITICAL: ipv4/ipv6, not addr/addr6)
      ue?: {
        ipv4?: string;   // e.g., "10.45.0.11"
        ipv6?: string;   // e.g., "2001:db8:cafe:0:7a3c:91ef:4b22:d8a1"
      };
      
      // SMF Addresses
      smf?: {
        ipv4?: string;
        ipv6?: string;
      };
      
      pcc_rule?: unknown[];
    }>;
  }>;
  
  schema_version: number;  // Always 1
  __v: number;             // MongoDB version key
}
```
