Top-Level Keys

Present in all service YAML files. The <nf> block name matches the service: mme, amf, smf, etc. Open5GS loads one YAML per process — in an all-in-one deployment every service block can live in the same file; in a split deployment each service gets its own file with only its own block plus logger/global.

KeyTypeDescription
loggerobjectLogging configuration
globalobjectGlobal settings shared across all NFs in the same process
db_uristringMongoDB URI, e.g. mongodb://localhost/open5gs. Also read from the DB_URI environment variable, which takes precedence when set
<nf>objectService-specific config block (mme, amf, smf, upf…)
logger:
Parsed in lib/app/ogs-init.c. Applies to whichever service loads this file — each process gets its own log target, so a split deployment typically has a different logger.file.path per YAML file.
YAML
logger:
  level: info                       # trace | debug | info | warn | error | fatal
  domain: mme                       # HIDDEN: filter logs to this domain name only
  file:                             # New format (preferred)
    path: /var/log/open5gs/mme.log
    timestamp: true                 # HIDDEN: prepend ISO timestamp to each line
  default:                          # HIDDEN: default output stream settings
    timestamp: false                # HIDDEN: suppress timestamp on stdout/default output

# DEPRECATED (still parsed, warns at runtime):
# logger:
#   file: /var/log/open5gs/mme.log   # scalar form deprecated in favor of file.path
KeyTypeDefaultExampleDescription
levelenuminfodebugVerbosity floor: trace < debug < info < warn < error < fatal. Drop to debug when chasing a NAS/NGAP decode issue; trace is extremely chatty and mainly used when developing Open5GS itself.
file.pathstring—/var/log/open5gs/amf.logAbsolute path to the log file. If omitted, Open5GS logs to stdout, which is the default in the official Docker images so container log drivers can capture it.
domainstring—gtpRestrict log output to a single internal logging domain (e.g. gtp, pfcp, nas, sbi). Useful for isolating one subsystem's chatter without wading through the rest at debug level.
file.timestampbooltruetruePrepend an ISO-8601 timestamp to each log line written to file.path. Turn it off if you're piping into a log shipper that already stamps arrival time and don't want the duplication.
default.timestampbooltruefalseSame as file.timestamp but for the default stdout/stderr stream instead of a file. Journald and most container runtimes already timestamp captured output, so disabling this avoids doubled timestamps in docker logs.
Deprecated form: logger: { file: /path } (a bare string instead of an object) still parses but logs a deprecation warning on startup. Use logger.file.path going forward.
global:
Parsed in lib/app/ogs-config.c. These settings apply to all services that load the same config file — in an all-in-one process that means every NF, in a split deployment it means whatever single NF that file's process runs. The parameter: block holds boolean feature flags — the most useful hidden options live here.

global.parameter — boolean feature flags

All entries in this table are hidden — none appear in the official sample configs, and the global.parameter block itself is also absent from samples. They're read directly from source (lib/app/ogs-config.c).
KeyTypeDefaultDescription
use_openairboolfalseOmits HashMME from the 4G Security Mode Command and the second byte of EPS Network Feature Support. Fixes attach failures (EMM Cause 24) against OpenAirInterface and some early 5G-capable UEs that don't implement HashMME verification per TS 24.301 §5.4.3.4. Set true only if you see repeated SMC rejects in MME logs from a specific UE model.
fake_csfbboolfalseMakes the MME answer SGs-AP CSFB procedures with synthetic success responses even though no real MSC/VLR is connected. Used to validate the LTE-side CSFB call flow in a lab without deploying a circuit-switched core.
use_upg_vppboolfalseSwitches the UPF data plane to UPG-VPP (a DPDK/VPP-based fast path) instead of the built-in kernel TUN-based forwarder. Set this when the UPF needs to sustain multi-Gbps throughput that the kernel path can't hit.
no_pfcp_rr_selectboolfalseDisables round-robin UPF selection in the SMF; the first reachable UPF matching the requested DNN is always chosen. Set this in single-UPF deployments to skip the scheduling step, or when one UPF should always be preferred (e.g. it has the better NIC).
no_time_zone_informationboolfalseStops MME/AMF from including the UE Time Zone IE in NAS Attach/Registration Accept. A handful of UE baseband stacks mishandle this IE and misreport local time; disabling it is a targeted workaround rather than something to flip by default.
no_ipv4v6_local_addr_in_packet_filterboolfalseOmits the local IPv4v6 address from PDR packet filters sent over N4/PFCP. Some third-party UPFs reject filter rules containing a local address; enable this as a compatibility workaround when pairing Open5GS SMF with a non-Open5GS UPF.
prefer_ipv4boolfalseWhen both an IPv4 and IPv6 address are available for the same peer, prefer the IPv4 one. Useful in dual-stack labs where IPv6 routing is present but not fully validated, to avoid flaky IPv6-first connection attempts.
no_ipv4boolfalseDisables IPv4 socket binding network-wide for the process. Set only in an IPv6-only deployment — combining this with an IPv4-only peer address in the same config will simply fail to connect.
no_ipv6boolfalseDisables IPv6 socket binding network-wide for the process. The common case: your host has an IPv6 link-local address that Open5GS would otherwise try to bind, and you want to force IPv4-only operation.
multicastboolfalseEnables multicast socket support on the NF's listening sockets. Relevant mainly for eNB/gNB discovery scenarios or lab setups that rely on multicast-based peer discovery rather than static addressing.
no_hss / no_mme / no_sgwc / no_sgwu / no_pgw / no_pcrf / no_nrf / no_scp / no_sepp / no_amf / no_smf / no_upf / no_ausf / no_udm / no_pcf / no_nssf / no_bsf / no_udrboolfalseSkips the internal service-instance counter/init for that NF. Used when several NF blocks share one config file but you only want a subset of them to actually start in this process — set no_<nf>: true for the ones to skip.

global.sockopt — socket tuning

Applies as the process-wide socket default. The identical key structure can be repeated as an option: sub-key under any specific server/client address entry (see the Notes section) to override just that one socket.

KeyTypeExampleDescription
no_delaybooltrueSets TCP_NODELAY on all TCP sockets, disabling Nagle's algorithm. Nearly always wanted for control-plane protocols like SBI/HTTP where message latency matters more than packing small writes.
sctp_nodelaybooltrueSets SCTP_NODELAY on S1AP/NGAP SCTP associations, avoiding Nagle-style batching on signaling messages. Leave on for realistic RAN signaling latency.
tcp_nodelaybooltrueAlias for no_delay, kept for naming symmetry with sctp_nodelay.
so_linger.l_onoffbooltrueEnables SO_LINGER so socket close() blocks until queued data is sent or the linger timer expires, rather than the default abortive close. Helps avoid truncated SBI responses under high connection churn.
so_linger.l_lingerint (s)10Linger duration in seconds when so_linger.l_onoff is set. Keep this small — a large value can stall worker threads waiting on slow clients.
so_bindtodevicestringeth1Binds all of this NF's sockets to a specific network interface name rather than by address, via SO_BINDTODEVICE. Useful on multi-homed hosts where the same IP might route out different interfaces.
sctp.spp_hbintervalint (ms)5000SCTP heartbeat interval for path liveness checks on S1AP/NGAP associations. Lower it in radio labs with flaky links to detect path failure faster; raise it to reduce signaling overhead on stable production links.
sctp.spp_sackdelayint (ms)200Delay before sending a SACK for received SCTP chunks. Lowering it trades a little bandwidth for lower perceived latency on S1AP/NGAP.
sctp.srto_initialint (ms)3000Initial SCTP retransmission timeout. Tune down on low-latency local networks to fail over to path retransmission faster.
sctp.srto_minint (ms)1000Minimum SCTP RTO — the floor the RTO estimator will not go below even on a very fast link.
sctp.srto_maxint (ms)60000Maximum SCTP RTO — the ceiling the RTO estimator won't exceed, bounding how long the stack waits before giving up on a path.
sctp.sinit_num_ostreamsint10Number of outbound SCTP streams requested at association setup. More streams reduce head-of-line blocking between unrelated S1AP/NGAP procedures on the same association.
sctp.sinit_max_instreamsint10Maximum number of inbound SCTP streams this NF will accept from the peer at association setup.
sctp.sinit_max_attemptsint4Number of INIT retransmission attempts before giving up on establishing the SCTP association.
sctp.sinit_max_init_timeoint (ms)8000Maximum timeout for the INIT handshake phase of SCTP association setup, independent of the per-attempt RTO.

global.pool — packet buffer pool sizes hidden

All entries here are hidden — the whole global.pool block is absent from official samples. Each key is a byte-size class; the value is how many pre-allocated buffer clusters of that size Open5GS reserves at startup. Under-sizing a class causes allocation fallbacks (and a logged warning) under load; over-sizing simply reserves more resident memory up front.

KeyTypeDefaultExampleTypically backs
128int10242048Tiny control messages — SBI header fragments, short Diameter AVPs, NAS IE fragments.
256int10242048Small signaling PDUs — typical S1AP/NGAP procedure messages, short GTP-C control messages.
512int10242048Mid-size signaling — NAS messages with several IEs, PFCP session establishment requests with multiple PDRs/FARs.
1024int10244096Larger control-plane payloads — bulk PFCP rule sets, larger SBI JSON bodies (subscription data, policy responses).
2048int1281024Small to mid data-plane packets — GTP-U packets under ~2KB, common for VoIP/signaling-heavy traffic mixes.
8192int128512Standard MTU-sized GTP-U payloads — the common case for regular internet-DNN user traffic at a 1400-byte UE MTU.
32768int128256Jumbo/reassembled GTP-U payloads — fragmented IP packets reassembled before encapsulation, or larger internal buffers.
bigint832The largest buffer class, for full-size GTP-U payloads and bulk transfers; increase cautiously since each cluster reserves the most memory per unit.

Shared time: Section

Appears under the service root block (e.g. mme.time, amf.time). Parsed in lib/app/ogs-config.c and shared across MME, AMF, SMF, and the 5G SBI-based NFs — only the timer sub-keys differ by NF family.

YAML
mme:                      # or amf, smf, any SBI-based NF
  time:
    nf_instance:
      heartbeat: 10           # SBI NF heartbeat interval (seconds)
      validity: 3600          # NF instance validity duration (seconds)
    subscription:
      validity: 86400         # NF subscription validity (seconds)
    message:
      duration: 5000          # Message response timeout (milliseconds)
    handover:
      duration: 10000         # Handover timeout (milliseconds)
    # MME-specific:
    t3402:
      value: 720              # Seconds; UE re-attach timer (default 720)
    t3412:
      value: 3240             # Seconds; periodic TAU timer (default 3240)
    t3423:
      value: 720              # Seconds; IMSI detach timer (default 720)
    # AMF-specific:
    t3502:
      value: 720              # Seconds; 5G registration retry timer
    t3512:
      value: 3240             # Seconds; 5G periodic registration timer
KeyTypeDefaultDescription
nf_instance.heartbeatint (s)10How often this NF sends an SBI heartbeat to keep its NRF registration alive. Shorten it in a lab to see failover/deregistration behavior faster; lengthen it in production to cut SBI chatter.
nf_instance.validityint (s)3600How long the NRF considers this NF's registration valid without a heartbeat before expiring it. Should be several multiples of heartbeat to tolerate transient misses.
subscription.validityint (s)86400Validity duration for SBI subscriptions this NF creates with other NFs (e.g. AMF subscribing to SMF status). Expired subscriptions must be renewed or re-created.
message.durationint (ms)5000Timeout for waiting on a response to an outbound SBI or protocol message before treating the peer as unresponsive.
handover.durationint (ms)10000Timeout budget for a handover procedure to complete before it's aborted and the UE falls back to the source cell/AMF.
t3402.valueint (s)720MME-only. After an attach reject, how long the UE (per its NAS timer) waits before retrying attach — mirrored here so the MME's internal timers stay consistent with what it tells the UE.
t3412.valueint (s)3240MME-only. Periodic Tracking Area Update interval given to the UE; shortening it increases signaling load but detects silently-dead UEs faster.
t3423.valueint (s)720MME-only. IMSI detach/deactivation guard timer.
t3502.valueint (s)720AMF-only, the 5G equivalent of T3402 — registration retry timer given to the UE after a registration reject.
t3512.valueint (s)3240AMF-only, the 5G equivalent of T3412 — periodic registration update interval.

Shared serving: Section

Appears under the root of 5G SBI-based NFs (most prominently NRF) to declare which PLMNs this instance serves. Parsed alongside the rest of the SBI context.

YAML
nrf:                      # most commonly seen on NRF
  serving:
    - plmn_id:
        mcc: "999"
        mnc: "70"
    - plmn_id:
        mcc: "310"
        mnc: "170"
KeyTypeExampleDescription
serving[].plmn_id.mccstring (3-digit)"999"Mobile Country Code this instance serves. 999 is the 3GPP test/reserved MCC Open5GS ships in its sample configs.
serving[].plmn_id.mncstring (2-3 digit)"70"Mobile Network Code paired with the MCC. Declare one entry per PLMN for multi-operator/multi-tenant NRF deployments — a second entry with mcc: "310", mnc: "170" lets the same NRF serve a roaming partner's PLMN too.

Diameter / FreeDiameter

MME supports only the freeDiameter: <path> form, pointing at a separate freeDiameter XML config. HSS, SMF, and PCRF support both that path reference and inline Diameter configuration keys directly under the service block.

YAML — inline form (HSS / SMF / PCRF)
hss:                        # or smf, pcrf
  identity: hss.open5gs.org  # Diameter Origin-Host
  realm: open5gs.org         # Diameter Origin-Realm
  port: 3868                 # Diameter port (default 3868)
  sec_port: 5868             # Diameter TLS port
  listen_on: 127.0.0.5       # Bind Diameter to this address
  no_fwd: false              # Disable Diameter forwarding agent
  tc_timer: 30               # Diameter Tc timer (seconds)
  load_extension:
    - module: /usr/lib/x86_64-linux-gnu/freeDiameter/dbg_msg_dumps.fdx
      conf: /etc/freeDiameter/dbg_msg_dumps.conf
  connect:
    - identity: mme.open5gs.org
      port: 3868
      tc_timer: 30
KeyTypeDefaultDescription
identitystring (FQDN)—Diameter Origin-Host advertised in every message this node sends — must be a resolvable-looking FQDN even in a lab, since some peers validate its shape.
realmstring—Diameter Origin-Realm, typically the domain suffix shared by all nodes in the deployment (e.g. open5gs.org).
portint3868TCP/SCTP port for plaintext Diameter.
sec_portint5868Port for Diameter over TLS, used when peers require encrypted transport.
listen_onIP—Bind address for the Diameter stack. On a multi-homed host, pin this rather than binding to all interfaces.
no_fwdboolfalseDisables this node's Diameter message-forwarding-agent behavior, so it only originates/terminates messages for itself rather than relaying for other realms.
tc_timerint (s)30Diameter Tc timer — the interval between reconnection attempts to a peer after a failed connection.
load_extension[].modulestring (path)—Path to a freeDiameter extension shared object, e.g. the built-in dbg_msg_dumps.fdx for wire-level Diameter tracing during debugging.
load_extension[].confstring (path)—Config file passed to the loaded extension module.
connect[].identitystring (FQDN)—Origin-Host of a Diameter peer to actively connect to on startup, rather than waiting for it to connect in.
connect[].portint3868Port to connect to on that peer.
connect[].tc_timerint (s)30Per-peer override of the reconnection interval for this specific connect entry.
mme:
Mobility Management Entity 4G
Parsed in src/mme/mme-context.c. The central 4G control-plane node — terminates S1AP from the eNB and NAS from the UE, drives GTP-C toward the SGW-C. Notable hidden options: hss_map for multi-HSS routing per PLMN, access_control for per-PLMN rejection, emergency for emergency numbers.
mme.yaml
mme:
  freeDiameter: /etc/freeDiameter/mme.conf

  relative_capacity: 100    # Load balancing weight (0-255, default 100)

  s1ap:
    server:
      - address: 127.0.0.2   # S1-MME bind address
        port: 36412          # S1AP port (default 36412)
        dev: eth0            # Alternative to address: bind to interface

  gtpc:
    server:
      - address: 127.0.0.2
    client:
      sgwc:
        - address: 127.0.0.3
      smf:
        - address: 127.0.0.4
      sgsn:                  # HIDDEN: Gn interface to 2G/3G SGSN
        - address: 10.10.0.20

  gummei:
    - plmn_id:
        mcc: "999"
        mnc: "70"
      mme_gid: 1
      mme_code: 1

  tai:
    - plmn_id:
        mcc: "999"
        mnc: "70"
      tac: 1

  access_control:            # HIDDEN: per-PLMN UE rejection
    - plmn_id:
        mcc: "310"
        mnc: "170"
        reject_cause: 11     # EMM cause: PLMN not allowed

  hss_map:                   # HIDDEN: route S6a to different HSS per PLMN
    - plmn_id:
        mcc: "310"
        mnc: "170"
        host: hss2.example.org
        realm: example.org

  security:
    integrity_order:         # EIA algorithm preference (tried in order)
      - EIA2                 # 128-EIA2 AES (recommended)
      - EIA1                 # 128-EIA1 SNOW 3G
      - EIA0                 # Null — no integrity
    ciphering_order:         # EEA algorithm preference
      - EEA0                 # Null — no encryption
      - EEA2                 # 128-EEA2 AES
      - EEA1                 # 128-EEA1 SNOW 3G

  network_name:
    full: "Open5GS Network"  # Broadcast to UEs in NAS messages
    short: "Open5GS"

  mme_name: open5gs-mme0

  time:                      # See shared time section (t3402/t3412/t3423)

  metrics:
    server:
      - address: 127.0.0.2
        port: 9090

  emergency:                 # HIDDEN: emergency call configuration
    dnn: internet
    number:
      - digits: "911"
        categories:          # police | ambulance | fire | marine | mountain
          - police
          - ambulance

  sgsap:                     # CSFB: SGs-AP interface to MSC/VLR
    client:
      - address: 127.0.0.88  # MSC/VLR address — use IP, not hostname (DNS at boot)
        local_address: 127.0.0.2
        port: 29118
        map:                 # TAI → LAI mapping
          tai:
            plmn_id:
              mcc: "999"
              mnc: "70"
            tac: 1
          lai:
            plmn_id:
              mcc: "999"
              mnc: "70"
            lac: 1
KeyTypeDefaultDescription
freeDiameterstring (path)—Path to the freeDiameter XML config used for the S6a interface to the HSS. This is the only Diameter form the MME accepts — unlike HSS/SMF/PCRF it does not support inline identity/realm keys.
relative_capacityint100Load-balancing weight (0-255) broadcast in the S1AP Setup Response. eNBs that implement MME load balancing steer more UEs toward higher-capacity values — raise it on a beefier MME instance in a multi-MME pool.
s1ap.server[].addressIP—Bind address for the S1-MME SCTP listener facing the eNB, e.g. 127.0.0.2 in the single-host demo topology.
s1ap.server[].portint36412SCTP port for S1AP.
s1ap.server[].devstring—Bind to a named interface (e.g. eth0) instead of a literal address — handy when the interface gets a DHCP address unknown at config time.
gtpc.server[].addressIP—MME's own GTP-C bind address, e.g. 127.0.0.2.
gtpc.client.sgwc[].addressIP—SGW-C GTP-C address the MME sends Create/Modify Session Requests to, e.g. 127.0.0.3.
gtpc.client.smf[].addressIP—SMF GTP-C address for direct S11-like signaling in combined 4G/5G deployments, e.g. 127.0.0.4.
gtpc.client.sgsn[].addressIP—Gn interface — GTP-C client address for a legacy 2G/3G SGSN, used only when supporting inter-RAT handover/fallback to GERAN/UTRAN.
gummei[].plmn_id.mcc/mncstring—PLMN this MME identity belongs to, e.g. mcc 999, mnc 70 (the Open5GS test PLMN).
gummei[].mme_gidint—MME Group ID (0–65535), part of the GUMMEI that uniquely identifies this MME to eNBs.
gummei[].mme_codeint—MME Code (0–255), the final component of the GUMMEI.
tai[].tacint—Tracking Area Code this MME serves for the given PLMN, e.g. 1.
access_control[].reject_causeint—Rejects UEs from the listed PLMN with a configurable EMM cause code (e.g. 11 = PLMN not allowed). Useful for MVNO gating in a shared-RAN lab, or for deliberately testing UE reject-cause handling.
hss_map[].plmn_id / host / realmobject—Routes S6a Diameter to a different HSS Origin-Host/Realm based on the attaching UE's home PLMN, so one MME can serve subscribers whose profiles live on separate HSS instances (e.g. an MVNO's own HSS at hss2.example.org).
security.integrity_orderlist (enum)EIA2, EIA1, EIA0Ordered NAS/AS integrity algorithm preference offered to the UE. Keep EIA0 (null) last and only present at all if you have a specific reason to allow unauthenticated-integrity UEs (e.g. certain emergency-call flows).
security.ciphering_orderlist (enum)EEA0, EEA2, EEA1Ordered ciphering algorithm preference. The upstream default actually prefers EEA0 (null) first for lab compatibility — flip the order to prefer EEA2 first in anything resembling production.
network_name.full / .shortstring—Operator network name strings broadcast to UEs in NAS messages and shown in the phone's carrier display.
mme_namestring—Free-text MME identity string used in logging and some Diameter contexts, e.g. open5gs-mme0.
metrics.server[].address/portIP / intport 9090Prometheus metrics endpoint for this MME instance.
emergency.dnnstring—DNN/APN used for the emergency PDN bearer established during an emergency attach.
emergency.number[].digitsstring—Emergency number string (e.g. "911", "112") the MME recognizes and advertises to UEs in Attach Accept for local emergency dialing.
emergency.number[].categorieslist (enum)—Service categories for that number: police, ambulance, fire, marine, mountain. Declare multiple categories per number when one emergency line covers several services.
sgsap.client[].addressIP—MSC/VLR address for the CSFB SGs-AP interface. Use a literal IP, not a hostname — DNS is resolved once at process boot, so a hostname that later re-resolves won't be picked up without a restart.
sgsap.client[].local_addressIP—MME's own bind address for the SGs-AP connection to the MSC/VLR.
sgsap.client[].portint29118SGs-AP port.
sgsap.client[].map.tai / .laiobject—Maps this MME's Tracking Area (TAI) to the 2G/3G Location Area (LAI) the MSC/VLR expects, required so CS paging can locate the UE correctly across the CSFB boundary.
hss:
Home Subscriber Server 4G
Parsed in src/hss/hss-context.c. Uses MongoDB for subscriber storage and speaks Diameter S6a to the MME. Shares the inline Diameter keys documented above.
hss.yaml
hss:
  # Inline Diameter config (or use freeDiameter: path)
  identity: hss.open5gs.org
  realm: open5gs.org
  port: 3868

  diameter_stats_interval: 60  # HIDDEN: log Diameter stats every N seconds

  sms_over_ims: false          # HIDDEN: enable SMS-over-IMS (SH interface)

  use_mongodb_change_stream: false
  # HIDDEN: use MongoDB change streams for real-time subscriber updates
  # without polling. Requires MongoDB replica set mode.

  metrics:
    server:
      - address: 127.0.0.5
        port: 9090
KeyTypeDefaultDescription
identity / realm / port——Standard inline Diameter identity, see the shared Diameter section above.
diameter_stats_intervalint (s)0 (off)Logs Diameter message-rate statistics every N seconds to help spot abnormal S6a traffic (retry storms, a misbehaving MME). Set to 60 when diagnosing HSS load.
sms_over_imsboolfalseEnables the SH interface behavior needed for SMS-over-IMS, where the HSS mediates SMS delivery info toward the IMS core rather than only classic circuit-switched SMS via MAP/SGs.
use_mongodb_change_streamboolfalseUses MongoDB change streams to push subscriber provisioning updates to the HSS in real time instead of polling. Requires MongoDB running in replica set mode — pointed at a standalone MongoDB instance, the HSS will error on startup when it tries to open the change stream.
metrics.server[].address/portIP / intport 9090Prometheus metrics endpoint, e.g. 127.0.0.5:9090.

Notes & Quirks

global.parameter placement

Important: All parameter: flags live under the global: block, not under the service block. Even in a service-specific YAML file like mme.yaml, the parameter must be at global.parameter.use_openair, not mme.parameter.use_openair.

apn vs dnn

In PFCP/session blocks, apn and dnn are accepted interchangeably. dnn is the current 5G name; apn is the 4G legacy alias. Both are parsed identically in lib/pfcp/context.c, so a mixed 4G/5G deployment can use whichever name reads more naturally in each file without any functional difference.

FreeDiameter path vs inline config

MME only supports the freeDiameter: /path/to/file.conf form. HSS, SMF, and PCRF support both the file reference and inline Diameter keys directly under the service block (identity, realm, connect, load_extension, etc.). If you're migrating an MME config's Diameter setup to one of those other services, you can either keep pointing at the same freeDiameter XML file or fold its contents into inline YAML keys — both work.

use_mongodb_change_stream (HSS)

Hidden HSS option: Enables MongoDB change streams for real-time subscriber provisioning updates without polling. Requires MongoDB in replica set mode — a standalone MongoDB instance will reject the change stream and the HSS will error on startup. If you're running a single-node MongoDB for a lab, either leave this off or convert that node to a single-member replica set first.

diameter_stats_interval (HSS / PCRF)

Hidden option present on both HSS and PCRF. Logs periodic Diameter message-rate statistics at the given interval in seconds; unset (or 0) means the stats logging is off. Turn it on temporarily (e.g. 60) when diagnosing S6a or Gx signaling load, then turn it back off — it adds log volume that's rarely useful in steady-state operation.

fake_csfb

When enabled, the MME sends fake SGs-AP responses without a real MSC/VLR. Lets you test CSFB call flows end-to-end on the radio side without commissioning a real circuit-switched core. The UE will see a voice call "succeed" at the LTE signaling layer but nothing happens at the application layer — useful for RAN interop testing, not for anything resembling a real CSFB deployment.

no_pfcp_rr_select

Disables round-robin UPF selection. With this set, SMF always picks the first reachable UPF that matches the DNN. Useful in single-UPF deployments to avoid any scheduling overhead, or when UPF #1 should always be preferred (e.g. it has better hardware or is closer on the network path).

no_time_zone_information

Prevents MME/AMF from including the UE Time Zone IE in NAS Attach/Registration Accept. Some devices behave poorly when this IE is present (misreporting local time or, in rarer cases, rejecting the Accept message outright) — this is a targeted per-device-model workaround rather than a setting to change proactively.

Socket option inheritance

The full sockopt key structure from global.sockopt (see global.sockopt) can also appear as an option: sub-key under any specific server or client address entry across every service — S1AP, NGAP, GTP-C, GTP-U, PFCP, SBI, SGs-AP all accept it the same way. This gives per-interface tuning (e.g. a shorter SCTP heartbeat just on the S1AP link to a flaky eNB) without changing the process-wide default that every other socket inherits.

Full Working Example — Combined 4G + 5G Core

Everything below is one coherent, deployable configuration exercising every NF this reference covers, wired together into a combined 4G EPC + 5G SA core running on one host — laid out the way Open5GS actually expects it: one YAML file per daemon, each under /etc/open5gs/. Every daemon only ever reads its own top-level block plus logger/global, so each file below is self-contained and independently deployable — copy the ones you need for your topology (e.g. skip scp.yaml/sepp.yaml entirely for a single-operator lab) and leave the rest out. What actually matters, and what trips people up, is which bind addresses have to leave loopback — that's the focus of this section.

Which IPs must be reachable from your LAN

Every official Open5GS sample config binds everything to 127.0.0.x loopback aliases, because in the demo topology every NF runs as a process on the same host and only ever talks to other processes on that same host. That stops working the moment a real eNB or gNB enters the picture: it's a separate physical (or virtual) box sitting elsewhere on your network, and loopback addresses are by definition unreachable from anywhere but the host itself. Only four bind addresses actually face the RAN — everything else is pure inter-process plumbing and can stay on loopback indefinitely, even in production, as long as those NFs stay co-located on one host.

Gotcha if you run 4G and 5G at once on one host: SGWU's S1-U listener and UPF's N3 listener both default to the same UDP port, 2152. If you point both at the same LAN IP, the second daemon will fail to bind with "address already in use." The fix is to give the RAN-facing interfaces two separate LAN IPs — e.g. add a secondary address to the same NIC (ip addr add 192.168.1.101/24 dev eth0) and dedicate one IP to the 4G RAN segment (S1AP + S1-U) and the other to the 5G RAN segment (NGAP + N3). That's exactly what the example below does.
